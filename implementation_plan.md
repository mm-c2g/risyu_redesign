## 1. プロジェクト基本方針

*   **目的**: 画像（全体像・授業フレーム）のUIデザインおよびインタラクションをWeb上で忠実に再現するプロトタイプの構築。
*   **成果物**: HTML / CSS / JavaScript を用いた静的Webサイト（ポートフォリオ掲載用）。
*   **デザイン**: ご提示いただいた画像の配色（パステルカラー、白背景、角丸コンポーネント）とレイアウトを忠実に再現。

## 2. 画面構成およびUIコンポーネント仕様

### 2.1 全体レイアウト（画像準拠）
*   **ヘッダー領域**: 
    *   年度・学期（例：2026年度 前期）、学生情報
    *   登録期間表示、学年・学期切り替えトグル（前期/後期）
    *   ログ表示エリア（履修・取り消しの履歴をテキスト表示）
    *   カテゴリ（専攻・芸術教養など）のカラー凡例
*   **時間割グリッド**: 
    *   月曜日〜土曜日の横並びカラム。各曜日間に余白（マージン）を設けて区別しやすくする。
    *   時限は1〜6限。
*   **時間外授業エリア**:
    *   グリッドの下部に配置。集中講義などの特殊な科目を横並びのカードで追加できるエリア。
*   **フッター（右下）**:
    *   「履修を確定する」ボタンの配置。

### 2.2 授業フレーム（コマ）の仕様
*   **単一コマの構成**:
    *   左上：オンライン授業マーク（Wi-Fiアイコン等）
    *   右上：単位数（例：2など）
    *   中央：授業名
    *   下部：担当教員名
*   **複数コマの結合**:
    *   2コマ以上連続する授業の場合、間の境界線をなくし、1つの繋がった大きなフレームとして描画する。
*   **空きコマのインタラクション**:
    *   通常時：空の枠。
    *   ホバー（カーソルオーバー）時：中央に「＋」アイコンが出現。
    *   クリック（タップ）時：「＋」が「×」に変わり、そのコマで履修可能な「気になる授業リスト（my時間割）」がポップアップ（ポップオーバー）として表示される。

## 3. 機能ロジック・インタラクション（JS実装要件）

*   **授業の追加・削除**: 空きコマからポップアップされた授業リストを選択すると、時間割にその授業が反映され、ログエリアに履歴が追加される。
*   **学期切り替え**: 前期・後期のボタン切り替えで、表示される時間割データが切り替わる。

## 4. データ構造例 (JSON)

UIを構築するための擬似データの構造例です。

```json
{
  "courses": [
    {
      "id": "c1",
      "name": "グラフィックデザイン VII",
      "instructor": "大木 (卓)",
      "credits": 2,
      "day": "Tue",
      "periodStart": 2,
      "periodLength": 2,
      "category": "major",
      "isOnline": false
    }
  ]
}
```

## 5. 今後の進め方 (Phase)

1.  **Phase 1 (静的マークアップ)**: 画像のレイアウトに従い、HTML/CSSでグリッド、ヘッダー、各種フレームの見た目を構築。
2.  **Phase 2 (インタラクション実装)**: ホバー時の「＋」表示、クリックでのポップアップリスト展開、複数コマの結合表示ロジックを実装。
3.  **Phase 3 (データ連携・ログ機能)**: 擬似データから時間割を描画し、追加・削除アクションによって画面とログが動的に更新されるようにする。
